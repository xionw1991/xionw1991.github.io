初级

  • 关于单方面技术
    • ARC / MRC 简单叙述一下
      OC中的内存管理机制是以引用计数的方式进行管理的,即当计数器的数值大于0的时候,对象存在, 如果计数器值等于0则,对象销毁. ARC的英文全称是Auto Reference Counting, 意思是自动引用计数, 表示由系统自动管理引用计数, 它有三种指针类型,strong(默认)\ weak\assign, strong和weak用于对象,assign用于基本数据类型. ARC的内存管理原则是:只有当有强指针指向对象时,对象才会存在,当没有强指针指向对象时,对象会被销毁. MRC的英文全称是Manual Reference Counting, 意思是手动引用计数, 即需要程序员手动来管理对象的引用计数.
    • 封装过什么控件, 简单叙述一下
    • 属性readwrite，readonly，assign，retain，copy，nonatomic各是什么作用
    • Get Set方法
    • id 是什么
    • 代理, Block, 循环引用
    • 通知的使用
  • 关于项目方面
    • 参与过什么项目, 负责什么
    • 项目中你参与的模块遇到过什么大的问题, 怎么解决的, 详细叙述
    • Git/SVN
